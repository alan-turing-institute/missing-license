## Missing license detected

This issue has been created to notify you that this repository (`{repo_name}`) does not appear to have a license.

Without a license, **no one is allowed to download, copy, modify, or use your code without your explicit permission** as the copyright holder ([source](https://choosealicense.com/no-permission/)) - even though the repository is public.

Since your repo is public, we assume you want to grant the general public some rights to use the code. The most common way to do this is to use an open source license. They divide into two types:

* [Permissive licenses](https://en.wikipedia.org/wiki/Permissive_software_license), roughly speaking, allow anyone to do anything with your code (including commercial use). The [BSD 3-clause license](https://opensource.org/license/BSD-3-clause) is a common choice, and a good default.
* [Copyleft licenses](https://en.wikipedia.org/wiki/Copyleft) similarly allow use of your code, but with extra conditions on when the user themself must use a copyleft license. This propagates open source in the software ecosystem. A good default copyleft license is the [AGPL license](https://www.gnu.org/licenses/agpl-3.0.en.html).

For non-code content, similar good defaults are [CC-BY](https://creativecommons.org/licenses/by/4.0) for a permissive license and [CC-BY-SA](https://creativecommons.org/licenses/by-sa/4.0/deed.en) for a copyleft license.

**Note that to change the license of an existing project, you need the permission of all copyright holders, i.e. everyone who has contributed. This is one reason to add a license early on.**

### What to do

#### 1. Choose a license

Here are some useful resources to help you choose:
- https://choosealicense.com/
- https://opensource.org/licenses
- https://www.turing.ac.uk/blog/open-source-software-why-it-matters-and-how-get-involved
- https://www.software.ac.uk/guide/choosing-open-source-license

#### 2. Add your license to the repository

Follow the GitHub [docs](https://docs.github.com/en/communities/setting-up-your-project-for-healthy-contributions/adding-a-license-to-a-repository) on how to add a license to your repository:

1. Go to the repository root
2. Click **Add file** → **Create new file**
3. Name the file `LICENSE`, `LICENSE.md`, or `LICENSE.txt`
4. Click **Choose a license template** and select one that fits your needs

If this repository is intentionally unlicensed, please close this issue with a comment explaining why so we can keep track. If your code is intentionally unlicensed to forbid use of your code, we still recommend adding a detectable restrictive license to make your choice clear.
